## 一、产品概述

### 1.1 产品定位
CCWT 是一个**专为 Claude Code CLI 设计的自托管多用户 Web 工作台**。它通过浏览器提供完整的终端交互能力，并集成项目文件管理、多终端标签页、会话历史回溯、内置 SOCKS5 代理以及离线语音输入，让开发者能够在任何设备上（PC、平板、手机）无缝进行 AI 辅助编程。

### 1.2 核心价值

**⭐ 多用户隔离 · 项目隔离（核心差异点）**
CCWT 从设计之初就以**多用户共享服务器**为默认场景，提供完整的用户级和项目级隔离机制：
- **用户级隔离**：每个 CCWT 用户拥有完全独立的 Claude 配置目录、认证凭据、会话历史及专属项目空间，多个用户共享一台服务器时互不干扰、互不可见。
- **项目级隔离**：同一用户的不同项目分别存放在独立的文件夹中，Claude Code 的上下文、会话记录按项目自动隔离，切换项目时环境变量、工作目录自动同步。
- **认证独立**：每个用户需独立完成 Claude OAuth 认证，认证凭据加密存储在各自专属目录中，一人登录不影响他人。

**其他核心价值**
- **部署极简**：单一 Go 二进制文件，内嵌前端静态资源，无需安装运行时或配置复杂环境。
- **全功能无损**：通过 PTY 技术 100% 还原 Claude Code CLI 原生体验，支持所有 `/slash` 命令、MCP 协议及交互特性。
- **全平台适配**：一套代码，自动适应桌面、平板及手机屏幕，提供一致的开发体验。
- **认证无忧**：内置一次性 SOCKS5 代理，解决远程服务器首次登录 Claude 时的 IP 漂移顾虑。
- **隐私优先的语音输入**：提供离线语音识别能力，无需依赖云端服务，保护用户隐私。

### 1.3 目标使用场景
- 在云服务器上部署 CCWT，团队多名成员各自通过浏览器登录，使用独立的 Claude 环境进行远程 AI 编程，彼此项目与配置完全隔离。
- 个人开发者在服务器上搭建专属环境，同时管理多个项目，每个项目的 Claude 上下文独立，随时随地在 PC 或手机继续未完成的编码工作。
- 使用平板或手机应急处理代码问题，通过语音输入快速描述需求，让 Claude Code 修复或生成代码。

## 二、核心需求：用户隔离与项目隔离（重中之重）

### 2.1 问题背景
Claude Code CLI 默认将用户配置、认证凭据、会话历史存储在 `~/.claude/` 目录中。若多个用户在同一台服务器上直接共享该目录，将导致以下严重问题：
- **认证污染**：所有用户共用同一份 OAuth Token，A 用户的认证操作可能覆盖 B 用户的凭据。
- **会话历史混淆**：所有用户的对话记录混在一起，无法区分，且可能泄露敏感信息。
- **配置冲突**：不同用户的个人偏好（如快捷键、MCP 服务器配置）相互覆盖。
- **项目上下文混乱**：Claude 在不同项目间的上下文记忆可能串扰，导致 AI 行为异常。

### 2.2 隔离目标
CCWT 必须实现**两个层面的完全隔离**：

| 隔离层级 | 隔离对象 | 隔离内容 | 实现方式 |
| :--- | :--- | :--- | :--- |
| **用户级隔离** | CCWT 用户之间 | Claude 配置目录、OAuth 认证凭据、全局会话历史、个人偏好设置 | 为每个 CCWT 用户创建独立的 `~/.ccwt/users/<username>/.claude/` 目录，并通过环境变量 `CLAUDE_CONFIG_DIR` 强制指定。 |
| **项目级隔离** | 同一用户的不同项目之间 | 项目文件、项目内 Claude 上下文、项目级 `.claude/` 配置 | 每个项目存放在用户专属的 `workspace/` 下的独立子目录中，Claude Code 自动识别项目根目录下的 `.claude/` 配置。 |

### 2.3 隔离效果验证标准
- **认证独立性**：用户 A 完成 Claude OAuth 认证后，用户 B 登录 CCWT 仍需自行完成认证，无法复用 A 的 Token。
- **会话历史不可见**：用户 A 无法在 CCWT 界面中看到用户 B 的任何对话历史记录。
- **文件系统隔离**：用户 A 的文件树根目录仅展示其专属 `workspace/` 下的内容，无法浏览或访问用户 B 的文件。
- **项目上下文独立**：用户 A 在项目 `app1` 中的 Claude 对话上下文，不会影响到同一用户在项目 `app2` 中的新对话。

## 三、功能需求清单

### 3.1 用户体系与认证

| 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- |
| **独立用户系统** | CCWT 维护独立的用户账户，不与 Linux 系统用户绑定。 | 必须 |
| **用户注册** | 支持开放注册（可配置关闭）。注册时需提供用户名、密码。 | 必须 |
| **用户登录** | JWT 认证机制，登录成功后颁发 Access Token 和 Refresh Token。 | 必须 |
| **用户数据隔离** | 每个用户在 `~/.ccwt/users/<username>/` 下拥有专属目录，包含 `.claude/` 配置目录和 `workspace/` 项目根目录。 | 必须 |
| **管理员角色** | 支持 `admin` 角色，可管理用户（禁用、删除）、配置系统参数。 | 建议 |
| **密码安全** | 密码使用 bcrypt 哈希加盐存储。 | 必须 |

### 3.2 用户与项目隔离机制（核心）

| 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- |
| **用户专属 Claude 配置** | 启动终端时，自动设置 `CLAUDE_CONFIG_DIR=~/.ccwt/users/<username>/.claude`，确保每个用户的 Claude 配置、认证凭据、历史记录完全隔离。 | 必须 |
| **用户专属工作区** | 每个用户的文件树根目录固定为 `~/.ccwt/users/<username>/workspace/`，用户无法访问其他用户的文件。 | 必须 |
| **项目自动识别** | 当用户在终端中 `cd` 进入某个项目目录时，Claude Code 自动读取该项目根目录下的 `.claude/` 配置，实现项目级规则隔离。 | 必须 |
| **多项目并行隔离** | 用户可同时打开多个浏览器窗口或终端标签页，分别连接不同项目，各项目的 Claude 会话上下文互不干扰。 | 必须 |

### 3.3 Web 终端模块

| 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- |
| **PTY 伪终端** | 后端为每个终端标签页创建独立的伪终端进程，运行用户指定的 Shell（默认 `bash`）。 | 必须 |
| **实时双向通信** | 前端通过 WebSocket 与后端 PTY 保持长连接，毫秒级同步输入输出。 | 必须 |
| **多终端标签页** | 支持新建、重命名、关闭、切换终端标签页。每个标签页对应一个独立的 PTY 会话。 | 必须 |
| **终端会话保活与重连** | 页面刷新或网络断开后，后端 PTY 进程继续运行；前端重连时自动恢复终端内容（基于回滚缓冲区）。 | 必须 |
| **终端尺寸自适应** | 前端监听容器尺寸变化，实时通知后端调整 PTY 行列数，确保 CLI 界面渲染正确。 | 必须 |
| **终端回滚缓冲区** | 每个终端保留至少 5MB 的输出历史，供前端重连时回放。 | 必须 |
| **原生功能完整性** | 无损支持 Claude Code 的 `/slash` 命令、`!` 终端命令、`Ctrl+R` 历史搜索、MCP 及 Hooks。 | 必须 |

### 3.4 项目与文件管理

| 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- |
| **可折叠项目文件树** | 左侧或抽屉式面板，以树状结构展示用户专属 `workspace/` 目录，支持展开/折叠文件夹。 | 必须 |
| **点击切换工作目录** | 单击文件树中的文件夹，当前活动终端自动执行 `cd` 命令切换到该路径。 | 必须 |
| **文件预览** | 支持常见文本文件（如 `.py`, `.js`, `.md`, `.json`, `.yaml` 等）的在线预览，带语法高亮。 | 必须 |
| **文件编辑与保存** | 使用 Monaco Editor 提供完整的代码编辑能力，支持保存到磁盘。 | 必须 |
| **基本文件操作** | 支持新建文件/文件夹、重命名、删除、复制路径。PC 端通过右键菜单，移动端通过长按菜单触发。 | 必须 |
| **文件上传与下载** | 支持从本地上传文件至当前目录，以及将文件下载到本地设备。 | 建议 |

### 3.5 Claude Code 会话历史浏览器

| 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- |
| **用户专属历史** | 仅读取当前用户 `~/.ccwt/users/<username>/.claude/projects/` 下的 JSONL 日志文件。 | 必须 |
| **按项目分组展示** | 根据日志来源项目，以时间线形式展示每个项目的对话历史。 | 必须 |
| **只读时间线** | 以对话气泡或列表形式渲染历史记录，清晰区分用户输入与 Claude 输出。 | 必须 |
| **关键词搜索** | 支持对历史记录内容进行全文检索。 | 建议 |
| **代码块快捷复制** | 历史记录中的代码块支持一键复制到剪贴板。 | 必须 |

### 3.6 认证辅助：内置 SOCKS5 代理

| 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- |
| **一次性认证代理** | 提供内置 SOCKS5 代理服务，仅在首次部署或某用户 Token 失效时由管理员手动开启，用于协助用户在服务器端完成 Claude Code 的 OAuth 设备码认证。 | 必须 |
| **代理开关与状态显示** | 管理员界面提供开启/关闭按钮，并显示当前代理地址（如 `SOCKS5://<服务器IP>:1080`）及复制功能。 | 必须 |
| **认证流程引导** | 提供图文说明，指导用户如何在本地浏览器中配置 SOCKS5 代理以完成认证。 | 必须 |
| **安全限制** | 代理默认仅允许本地回环地址连接，或通过防火墙规则限制访问来源 IP，防止被滥用。 | 必须 |
| **自动关闭建议** | 认证完成后提示关闭代理，避免长期开启带来的安全风险。 | 建议 |

### 3.7 快捷指令面板

| 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- |
| **全局命令唤起** | PC 端通过 `Cmd/Ctrl + K`，移动端通过悬浮按钮或手势唤起命令面板。 | 必须 |
| **内置命令集** | 至少包含：新建终端、关闭当前终端、清屏、切换文件树显示、打开设置、语音输入开关等。 | 必须 |
| **命令搜索** | 支持输入关键词快速筛选命令。 | 建议 |

### 3.8 系统状态与配置

| 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- |
| **状态栏系统监控** | 实时显示服务器 CPU 使用率、内存占用，可选显示当前 Git 分支。 | 建议 |
| **项目环境变量注入** | 支持读取项目根目录下的 `.ccwt.env` 文件，在启动 PTY 时注入环境变量。 | 建议 |
| **外观主题** | 提供深色/浅色主题切换，并支持跟随系统设置。 | 必须 |

### 3.9 跨端自适应 UI

| 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- |
| **响应式布局** | 使用 CSS Grid/Flexbox 及媒体查询，自动适配桌面（≥768px）与移动端（<768px）布局。 | 必须 |
| **移动端侧边栏** | 文件树默认隐藏，通过汉堡菜单滑出，支持手势滑动关闭。 | 必须 |
| **移动端终端交互** | 处理虚拟键盘弹出导致的视口变化，避免遮挡终端；提供虚拟功能键栏（Ctrl, Tab, Esc, ↑, ↓ 等）。 | 必须 |
| **触摸友好设计** | 所有可交互元素尺寸不小于 44x44pt，间距充足；右键菜单替换为长按菜单。 | 必须 |

### 3.10 语音输入（离线优先）

| 功能点 | 详细描述 | 优先级 |
| :--- | :--- | :--- |
| **离线语音识别** | 后端集成 `whisper.cpp`，用户可通过前端录制音频并上传，由后端调用本地模型完成语音转文字。 | 必须 |
| **模型管理** | 首次使用时自动下载轻量级中文模型（如 `ggml-base.bin`），存放于 `~/.ccwt/models/`，所有用户共享。 | 必须 |
| **浏览器语音输入（备选）** | 前端同时提供 Web Speech API 入口，在支持的浏览器（如 Chrome Android）上可快速调用，作为离线方案的补充。 | 建议 |
| **隐私保护** | 离线模式下，用户音频数据仅在内存中处理，不落盘、不上传第三方。 | 必须 |

## 四、用户界面与交互设计

### 4.1 设计原则
- **专注内容**：终端是绝对核心，任何辅助 UI（文件树、菜单）均应可隐藏，减少对编码视线的干扰。
- **跨端一致性**：功能在不同设备上保持一致，交互方式符合平台直觉。
- **轻量现代**：界面简洁、加载迅速，避免繁重动画。

### 4.2 桌面端布局（宽度 ≥ 768px）

```
+-----------------------------------------------------------------------+
|  [汉堡菜单]  CCWT        [代理开关] [语音输入] [主题切换] [用户头像/设置] |
+---------------------------+-------------------------------------------+
|                           |  [终端1] [终端2] [终端3]  [+]               |
|       项目文件树           +-------------------------------------------+
|      （可折叠/展开）        |                                           |
|                           |                                           |
|      > workspace/         |              主终端显示区                   |
|        > my-app/          |             (xterm.js)                    |
|          src/             |                                           |
|          package.json     |                                           |
|        > another-project/ |                                           |
|                           |                                           |
|                           |                                           |
+---------------------------+-------------------------------------------+
| 状态栏:  CPU 5% | MEM 128MB | 当前项目: /workspace/my-app | 代理: 关闭  |
+-----------------------------------------------------------------------+
```

### 4.3 移动端布局（宽度 < 768px）

**默认视图（终端全屏）**
```
+-----------------------------------+
| [三]  CCWT      [🎤] [代理]  [:] |
+-----------------------------------+
| [终端1] [终端2] [终端3]      [+]  |   ← 标签栏可横向滑动
+-----------------------------------+
|                                   |
|                                   |
|            终端显示区               |
|           (xterm.js)              |
|                                   |
|                                   |
|                                   |
+-----------------------------------+
| CPU 5% | MEM 120M | my-app        |
+-----------------------------------+
```

**语音输入交互**
- 点击顶部麦克风图标 `[🎤]`，弹出录音面板，显示波形动画。
- 录制完成后，前端检测浏览器是否支持 Web Speech API。若支持，优先使用浏览器识别；否则将音频文件上传至后端，调用 `whisper.cpp` 识别。
- 识别结果自动填入当前终端输入行，或直接发送。

### 4.4 核心交互流程示例

**场景一：新用户注册并完成 Claude 认证**
1. 用户访问 CCWT 登录页，点击“注册”，输入用户名、密码完成注册。
2. 登录后进入主界面，点击顶部“代理”开关（需管理员预先开启或用户自身为管理员），界面显示 `SOCKS5://server-ip:1080`。
3. 用户在自己的 PC 浏览器中配置该 SOCKS5 代理。
4. 返回 CCWT，新建终端，输入 `claude`，终端显示 OAuth 设备码认证链接。
5. 用户复制链接，在配置好代理的浏览器中打开，完成授权。
6. 授权成功后，终端自动进入 Claude Code 交互界面。用户关闭代理。
7. 该用户的 Claude 认证凭据永久保存在其专属的 `~/.ccwt/users/<username>/.claude/credentials.json` 中，后续登录直接可用。

**场景二：手机端语音编程**
1. 用户通过手机浏览器登录 CCWT，进入主界面。
2. 点击终端使其获得焦点，再点击顶部麦克风图标。
3. 说出“帮我写一个 Python 的日志工具类”，录音完成后自动识别并填入终端。
4. 用户点击虚拟键盘的“发送”键，Claude 开始生成代码。
5. 整个过程无需大量打字，适合移动场景。

### 5.5 关键模块设计要点（含隔离实现）

| 模块 | 核心职责 | 隔离实现要点 |
| :--- | :--- | :--- |
| **用户管理** | 注册、登录、JWT 签发与验证。 | 用户创建时，自动在 `~/.ccwt/users/<username>/` 下创建 `.claude/` 和 `workspace/` 目录，权限 `0700`。 |
| **用户隔离执行器** | 为每个用户的终端进程设置独立的环境变量。 | 启动 PTY 时，设置 `CLAUDE_CONFIG_DIR=~/.ccwt/users/<username>/.claude` 和 `HOME=~/.ccwt/users/<username>`，确保 Claude CLI 的所有读写操作限定在用户专属目录内。 |
| **文件系统服务** | 提供安全的文件/目录操作 API。 | 所有路径基于当前用户的 `workspace/` 进行解析和前缀校验，禁止访问 `../` 等越界路径。 |
| **会话历史解析器** | 读取 Claude Code 的 JSONL 日志。 | 仅读取当前用户 `~/.ccwt/users/<username>/.claude/projects/` 下的文件，按项目路径分组返回。 |
| **语音识别** | 提供离线语音转文字能力。 | 音频处理在内存中进行，不落盘；模型文件所有用户共享，存放于 `~/.ccwt/models/`。 |

## 六、非功能性需求

| 类别 | 要求 | 说明 |
| :--- | :--- | :--- |
| **性能** | 终端交互延迟 < 50ms；文件树加载 1000 个文件 < 500ms；语音识别延迟 < 2s。 | 通过 WebSocket 长连接、前端虚拟化和高效的 `whisper.cpp` 保证流畅体验。 |
| **可靠性** | WebSocket 断线自动重连；PTY 进程崩溃不影响其他标签页。 | 实现健壮的会话管理和错误恢复机制。 |
| **安全性** | 文件操作限制在用户工作区内；密码 bcrypt 哈希存储；JWT 签名验证；HTTPS 支持。 | 推荐配合 Nginx/Caddy 反向代理启用 TLS。 |
| **隔离性** | 用户间配置、项目、历史记录完全不可见；项目间 Claude 上下文独立。 | 通过目录权限（`0700`）和环境变量强制隔离。 |
| **兼容性** | 支持 Chrome/Firefox/Safari/Edge 最近两个主要版本。移动端支持 iOS Safari 和 Android Chrome。 | 响应式设计确保各端体验一致。 |
| **可维护性** | 代码结构清晰，核心模块低耦合。 | 后端采用分层架构，前端采用组合式 API 与 Pinia 模块化。 |

## 八、附录：术语表

| 术语 | 解释 |
| :--- | :--- |
| **PTY** | 伪终端（Pseudo Terminal），用于模拟真实的终端设备，支持交互式命令行程序。 |
| **SOCKS5** | 一种网络代理协议，可将流量通过中间服务器转发，此处用于统一 Claude 认证时的出口 IP。 |
| **OAuth Device Flow** | 一种为输入受限设备（如电视、服务器）设计的授权流程，用户通过另一设备浏览器完成登录。 |
| **JSONL** | JSON Lines 格式，每行一个独立的 JSON 对象，Claude Code 使用该格式存储对话日志。 |
| **whisper.cpp** | OpenAI Whisper 模型的 C++ 移植版，可在 CPU 上高效运行，实现离线语音识别。 |
| **Pinia** | Vue 官方推荐的状态管理库，与 Composition API 配合使用。 |
| **CLAUDE_CONFIG_DIR** | Claude Code CLI 的环境变量，用于指定配置目录，是实现用户隔离的核心机制。 |
